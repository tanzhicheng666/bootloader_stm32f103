* Basic Information:
	Core: Core files extracted from the SDK for easy modification;
	Bsp: Hardware driver layer. When changing the MCU, only this layer needs to be rewritten;
	     the application layer does not need modification.
	App: Application layer. Handles application-level code. All hardware operations must
	     go through the BSP layer interfaces only.
	Common: Common code, including basic data type redefinitions, etc.
	Algorithm: Algorithm code.
	Main: Main function files;
	Project: Project files, including target binary files;
	SDK: CMSIS and HAL libraries, containing only the libraries actually used.
	     When adding other peripherals, the corresponding libraries must be added.

	Clean.bat: Cleans up files generated by the project to reduce size for easy copying;
	Readme.txt: Project description file;

* Project Description:
	template;

* Note:
	This project example only includes the HAL libraries used by the code.
	When adding other peripheral features, the user needs to add the corresponding
	HAL libraries or use the project template (which includes all HAL libraries).
